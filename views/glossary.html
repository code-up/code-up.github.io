Glossary

Access Modifiers

An access modifier is one of public, protected or private. They can be given to methods, variables (incl. fields), classes, interfaces and constructors.

If no access modifier is given, this is the fourth access modifier: default. See the entries for public, protected, private and default access for more info.

Class

A class is a blueprint for an object. In a class you describe fields, methods and the special constructor.

Constructor

A special type of method which does not declare a return type, and has the same name as the class it's in. These methods are used to create an instance of the object whose blueprint is the class.

Default Access

The default access modifier (also known as package-private), is the access level given when the keyword is omitted. Package-private access allows members of the same package to access the item (item can be method, variable, constructor or interface), but not sub-classes or any other class (the world).

Field

A field is a variable, which has special status within a class. Fields are declared in the same way as variables, with the same access modifiers. A field will be remembered by the computer for as long as the object that created it is.

Keywords

A keyword is any reserved word within the language you're using. In other words, a keyword is any member of the language syntax. Within this glossary, all Java keywords have been highlighted like this: public, private and interface.

Variable

A variable is a named piece of information. Variables are given a type and a value, which the computer will remember until you don't need it anymore. Variables are also given an access modifier, and can be marked as final.
