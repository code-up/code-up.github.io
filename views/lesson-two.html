Lesson 2 - Languages

Programming languages can be incredibly different. Some are complicated, and let you take control of every low-level detail - which can be risky, but can also cause your programs to be efficient. Others will sacrifice performance in order to make your life easier.

Very broadly speaking, there are two main categories of languages. Functional languages, and Object-Oriented languages. Functional languages are often very basic, and are normally used to get a simple job done quickly. They use methods and variables, but there is no such thing as a type, so it's easy for errors to occur. Often, functional languages don't require compiling before being executed - and often exist as a 'script'.

Some Examples of Functional Languages
  • Bash - a basic Linux/Mac language.
  • Batch - a basic Windows language.

Object-Oriented languages are completely different. They are considered far more powerful than functional languages, because they allow better organisation of code, promote more reusable code, and are easier to extend.

What do you mean by Extend?

If you ask any programmer whether the current project they're working on does everything that was originally required of it, and nothing more - they'll say No. If they says Yes, they're fucking liars. Requirements change, and so we need to develop our projects in such a way that if we need to make it do something new, it's easy for us to change it. We'll explain much more about this later on.

Some Examples of Object-Oriented Languages
  • Java - a multi-platform language.
  • C++ - a complicated, but also multi-platform language.
  • C# - a simplified Windows version of C++.
  • Objective-C - a simplified Mac only version of C++.

These categories are a good way to distinguish and generalise between languages, but many languages blur the lines between them. I've put the most important ones below.

Examples of Hybrid Languages
  • Javascript - a language which forms one of the main languages of the Web. At it's core, javascript is OO (object-oriented), but it's often written in a functional style.
  • VBA - a Windows language, which Excel Macros are written in.
  • HTML - an XML based language.
  • CSS - a Web stylesheets language, which has some OO concepts.

What makes a Language Object-Oriented?

Object-Oriented languages (predictably) use the concept of an Object at their core. Objects, just like in real life, have properties. Take an Apple for example; Apples have a colour, a size, as well as many other properties which we can use to differentiate between two apples. Below, I'll describe an Apple with some properties in Java code.

public class Apple { private final String colour; private final int size; private final boolean edible; ... }

The code above describes our version of an Apple object; we have a text property called colour, an integer which describes the size, and a boolean which will tell us whether the variable is true or false. So far, I've called these variables 'properties', because we should consider them descriptive properties of the object we wish to describe. Unfortunately, in programming, the correct term for an object's properties are its 'fields'.

WTF does Public, Private, Final and Class mean?

The Public, Private, Final and Class are parts of the Java syntax. Public and Private are Access Modifiers - we'll go into these in detail later.

The Final keyword states that the method or field will never change. This allows us to create immutable objects - something I will definitely explain properly later.

Lastly, the Class keyword states that the following block of code describes a new type of object. Classes in Java must exist within their own file, where the name of the file matches the name of the Class exactly - no exceptions.

Apart from fields, objects also have methods. Methods, as described in the previous lesson, are named pieces of work, which can accept arguments and return data.

The fields of an object contain the data for that object; whereas the methods of an object, perform actions on that data.

Going back to our Apple, an example method could be to 'bite' the apple. For this to make sense, we'll add another field to our object; 'amountEaten'.

public class Apple { private final String colour; private final int size; private final boolean edible; private int amountEaten = 100; // Integer representing the percentage of the apple remaining. public void bite(int amountToBite){ this.amountEaten -= amountToBite; } }

A couple things are going on here. Firstly we declared our new integer field, amountEaten; note that because we intend to change this field, we do not mark it final.

Secondly, we provided a comment explaining the field. Comments are a way to give (often invaluable) information about a block of code, without affecting the compilation of that code. In fact, the compiler will delete all data within comments immediately. In Java, there are two types of comments; in-line comments // Which use the double forward slash syntax, and multi-line comments:

/** * Which use this odd asterix format. * */

Next, we described our method; first we marked it public, which will allow other Objects to use the method (more on this later). We then specified the method return type, in this case nothing or void. Next we named the method bite, and specified that it has one argument int amountToBite.

In the method body, we use a little bit of syntax magic. We could have written the line like this:

this.amountEaten = amountEaten - amountToBite;
instead, the Java syntax allows us to write the same line using the -= syntax.

The -= syntax effectively states 'subtract the next value from the variable.' The other 'number' based syntax features are integer++, which will add one to the specified variable (this is useful for counting). Both of these types of syntax work with plus and minus (i.e. += or integer--).

The Java syntax has several features, such as the above, which make our lives as developers a little easier. This is called Syntactic Sugar; as always, this will be discussed properly later.
